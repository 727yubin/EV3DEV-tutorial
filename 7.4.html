try - except

One final statement/loop is the try - except. This is the format:

try: do_something except optional_condition: do_something_else else: do_a_third_thing finally: do_a_fourth_thing

The else and finally will not always be there, but try and except always will be. The program will try whatever is in try. If there’s no problem, no harm done. If there is a problem, it will run what is in except. This is very useful, because if a program runs into an error without a try - except it will crash. Let’s see an example.

>>> try: ... 1 / 0 ... except: ... print("You can't divide by zero!") ... You can't divide by zero! >>> try: ... 1 / 1 ... except: ... print("You can't divide by zero!") ... 1.0

How does this work? Well, let’s see what happens when we just divide 1 by 0.

>>> 1 / 0 Traceback (most recent call last): File "<stdin>", line 1, in <module> ZeroDivisionError: division by zero

In the first case, because dividing by zero gives an error, the program runs what is in except. In the second case, 1 / 1 will not give an error, so the program ignores what is in except.

If there were no try - except, the program would have crashed.

In a try - except, it is better to mention the exact error in the except. The better code would read: >>> try: ... 1 / 0 ... except ZeroDivisionError: ... print("You can't divide by zero!")
