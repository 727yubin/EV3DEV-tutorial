Commenting code

When you program, there will be other people that read your code. That is why it is good practice to add comments to your code. Comments are ignored by the computer, and are meant to be read by other human programmers.

To add comments in Python code, put a # in front of the comment.

Let’s see how it works.

Open gedit, type the following, and save it as print.py. Overwrite if necessary.

1 #!/usr/bin/python3 2 print(“Hello, World!”) # Writing Hello, World! to the screen 3 print(“This is my 3rd python program.”) 4 #print(“This line will be ignored by the program.”)

Now let’s see what it gives.

$ python3 print.py Hello, World! This is my 3rd python program.

Let’s look at what happened.

Line 2 does what it’s meant to ; it prints Hello, World! to the screen. Notice how everything after the # is ignored until the next line.

At the beginning of line 4 there is a #. The computer just ignored it and moved on. Comments can sometimes be used to purposefully make the computer ignore a line. This is especially useful if you want to focus on part of the code.

What would happen if you delete the # at the beginning of line 4?

As a final note, bad comments are worse than no comments. When you change your code, make sure that the comments don't contradict the code. Also, don't comment obvious things, as they tend to clutter the screen and worsen readability. As an example, the comment in line 2 would be a bad comment.
