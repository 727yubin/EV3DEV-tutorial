2. Variables

Okay, so doing simple math, printing, and commenting was fun, but… it can't really do anything. It’s a dumb program. What’s the use of such a program?

For programs not to do the exact same thing over and over, we use variables.

Variables can contain an integer. An integer is a whole number that can be positive or negative.

>>> number = 5 >>> number + 2 7

What happened? In the first line, we make a variable called number, and we store the number 5. Then we add 2 to it, which gives the expected answer of 7.

Try doing number = 3 then number + 2. What do you notice?

Variables can also contain a string. A string is a group of characters.

>>> name = “John” >>> print(name) John >>> print(“Hello“, name) Hello John

Neat! Notice that we have to put quotation marks around the string to store it, but we do not need them for numbers. When using a variable, we do not put the variable’s name inside quotation marks.

We can have multiple variables:

>>> food = “bread” >>> drink = “water” >>> print(food, water) bread water

There are also things called booleans: True and False.

>>> state = True >>> print(state) True

Booleans can be especially useful. Be careful: when you put a boolean, you have to capitalize. You will get an error if you do not capitalize:

>>> state = false Traceback (most recent call last): File "", line 1, in NameError: name 'false' is not defined

So don’t forget to capitalize.

As a final note, there are a few restrictions on variable naming. Let’s see some examples.

>>> 123state = None File "", line 1 123state = None ^ SyntaxError: invalid syntax >>> state@ = None File "", line 1 state@ = None ^ SyntaxError: invalid syntax >>> class = None File "", line 1 class = None ^ SyntaxError: invalid syntax

By the way, giving a variable None means creating a variable with nothing in it. You can store None in a variable to have it as a placeholder until you put something else in it.

The name of a variable cannot start with a number. It can start with a letter then have a number, but it cannot start with a number. So state123 would be a valid variable name, while 123state wouldn’t.

Also, variable names cannot have a special character.

Fair enough. But what’s wrong with class? class is actually one of 33 python keywords, meaning that they cannot be used as a name of a variable. The list is here:

False elif lambda None else nonlocal True except not and finally or as for pass assert from raise break global return class if try continue import while def in with del is yield

Note: Keywords may change by version. Run import keyword; keyword.kwlist to check.

There’s no need to memorize this, as it will be fairly obvious by the time you get the hang of python.
